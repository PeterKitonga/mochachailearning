Nodejs In-depth Unit Testing with Mocha, Chai

This repo serves as a showcase of my learning process and knowledge on unit testing with mocha, chai. In this section I am briefly introduced to Mocha, Chai and how to set it up.

Mocha Setup

To setup mocha we first installed it globaly via the command npm install -g mocha. This installs a CLI that will be used to run our tests. With this in place we can start a new nodejs project via the npm init command. Thereafter we create a test folder in the project directory to house all the test files and configurations. Consider the below file(test/unit/bar.spec.js) as our spec. The below file uses the default nodejs assertion library. 

  1. const assert = require('assert');
  2.  
  3. /**
  4.  * 'describe' defines the suite of specs in the unit test
  5. */
  6. describe('file to be tested', () => {
  7.     /**
  8.      * 'context' defines the function being tested
  9.     */
  10.     context('function to be tested', () => {
  11.         /**
  12.          * 'before' defines the 'setup' in our tests
  13.          * used for setting up common variables, functions before any spec runs in the relative suite
  14.         */
  15.         before(() => {
  16.             console.log('================== before');
  17.         });
  18.  
  19.         /**
  20.          * 'after' defines the 'teardown' in our tests
  21.          * used for clean up after all specs in the relative suite run
  22.         */
  23.         after(() => {
  24.             console.log('================== after');
  25.         });
  26.  
  27.         /**
  28.          * 'beforeEach' & 'afterEach' work the same way as the above setup & teardown
  29.          * difference is they run before and after each spec
  30.         */
  31.         beforeEach(() => {
  32.             console.log('-------beforeEach');
  33.         });
  34.  
  35.         afterEach(() => {
  36.             console.log('-------afterEach');
  37.         });
  38.  
  39.         /**
  40.          * 'it' defines a spec testing something in the function
  41.         */
  42.         it('should do something', () => {
  43.             assert.equal(1, 1);
  44.  
  45.             /**
  46.              * IMPORTANT: it is good practice to always check the environment mode
  47.              * since it is not desired to run tests in production. This can be dangerous
  48.              * since we can drop database tables during tests which is not ideal for production.
  49.             */
  50.             if (process.env.NODE_ENV === 'development') {
  51.                 console.log(`ENV MODE: ${process.env.NODE_ENV}`);
  52.             }
  53.         });
  54.  
  55.         it('should do something else', () => {
  56.             assert.deepEqual({name: 'joe'}, {name: 'joe'});
  57.         });
  58.  
  59.         /* this is a pending spec */
  60.         it('this is a pending test');
  61.     });
  62.  
  63.     /**
  64.      * we can also have multiple contexts in a suite
  65.     */
  66.     context('another function', () => {
  67.         it('should do something');
  68.     });
  69. });
Chai Setup

Chai is an assertion library for node and can be used by any testing framework. It is commonly used with mocha and provides an extensive amount of configurability and assertion styles. In this setup I have used the 'expect' assertion style which complements mocha very well. Before using it we install it in our project first via the command npm install --save-dev chai. Consider the below spec file(test/unit/foo.spec.js) to demonstrate using chai as the assertion library. 

  1. const { expect } = require('chai');
  2.  
  3. describe('chai test', () => {
  4.     it('should compare some values', () => {
  5.         expect(1).to.equal(1);
  6.     });
  7.  
  8.     it('should test some other stuff', () => {
  9.         /**
  10.          * 'deep' is used when testing reference types(objects/arrays)
  11.         */
  12.         expect({name: 'foo'}).to.deep.equal({name: 'foo'});
  13.         /**
  14.          * 'property' is used when testing if an object property exists
  15.         */
  16.         expect({name: 'foo'}).to.have.property('name').to.equal('foo');
  17.         /**
  18.          * used when testing booleans(false/true)
  19.         */
  20.         expect(5 > 8).to.be.false;
  21.         expect(0 < 1).to.be.true;
  22.         /**
  23.          * 'a' can be used to test the data type of a variable/value
  24.         */
  25.         expect({}).to.be.a('object');
  26.         expect('foo').to.be.a('string');
  27.         expect(3).to.be.a('number');
  28.         /**
  29.          * here, we can chain multiple expectations
  30.         */
  31.         expect('bar').to.be.a('string').with.lengthOf(3);
  32.         expect([1, 2, 3].length).to.equal(3);
  33.         /**
  34.          * used when testing if a variable/value is null
  35.         */
  36.         expect(null).to.be.null;
  37.         /**
  38.          * used when testing if a variable/value is undefined
  39.         */
  40.         expect(undefined).to.not.exist;
  41.         /**
  42.          * used when testing if a variable/value is not undefined
  43.         */
  44.         expect(1).to.exist;
  45.     });
  46. });
Run Setup

First we setup a script in the package.json file like so: 

  1. "scripts": {
  2.     "test": "cross-env NODE_ENV=development mocha test/**/*.spec.js"
  3.  }

To run the tests above, enter the command: 

  1. npm test
