Basics of Testing

In this section, I learnt how to setup basic tests for functions, promises using chai assertions. I also got to learn about setting up test doubles for the unit tests. A test double can be described as an object that can stand in for a real object in a test. Simply put, it is similar to how a stunt double stands in for an actor in a movie. There are five common types of test doubles namely:

  1. Dummy object
  2. Fake object
  3. Stubs
  4. Mocks
  5. Spies
For further information, read about it here.

Testing a function

Here, I first created a function called 'add' in an example script(src/lib/demo.js). In a spec file(src/test/specs/demo.spec.js), I added a context for the function and added a spec for the function. 

  1. exports.add = (a, b) => {
  2.     return a + b;
  3. }
  1. describe('demo.js', () => {
  2.   context('add()', () => {
  3.     it('should add two numbers', () => {
  4.       expect(add(1, 2)).to.equal(3);
  5.     });
  6.   });
  7. });
Testing a callback function

Here, I tested a callback function called 'addCallback' in the example script(src/lib/demo.js). I passed the done() callback to the spec for this context to let mocha know that the callback has finished executing. Without invoking the done callback the test would timeout. 

  1. exports.addCallback = (a, b, _callback) => {
  2.     setTimeout(() => {
  3.         return _callback(null, a + b);
  4.     }, 20);
  5. }
  1. describe('demo.js', () => {
  2.   context('addCallback()', () => {
  3.     /**
  4.      * Just like in jasmine we pass a done callback as a param to let mocha know
  5.      * the function has finished running
  6.      */
  7.     it('should add two numbers with a callback', (done) => {
  8.       /**
  9.        * By default nodejs returns an err param in callbacks
  10.        * which is passed first
  11.        */
  12.       addCallback(1, 2, (err, result) => {
  13.         expect(err).to.not.exist;
  14.         expect(result).to.equal(3);
  15.         done();
  16.       });
  17.     });
  18.   });
  19. });
Testing promises

Here, I tested a promise called 'addPromise' in the example script(src/lib/demo.js). I used 4 different ways to test the promise:

  1. Using the done() callback
  2. Using return
  3. Using async/await
  4. Using the chai-as-promised plugin for chai

  1. exports.addPromise = (a, b) => {
  2.     return Promise.resolve(a + b);
  3.     // return Promise.reject(new Error('Fake Error'));
  4. }
  1. // import plugin
  2. const chaiAsPromised = require('chai-as-promised');
  3.  
  4. /**
  5.  * Plugins are extended into chai using the .use() method once imported
  6.  */
  7. chai.use(chaiAsPromised);
  8.  
  9. describe('demo.js', () => {
  10.   context('addPromise()', () => {
  11.     it('should add two numbers with a promise', (done) => {
  12.       addPromise(1, 2).then(result => {
  13.         expect(result).to.equal(3);
  14.         done();
  15.       }).catch(err => {
  16.         done(err);
  17.       });
  18.     });
  19.  
  20.     /**
  21.      * promises can also be returned without passing the
  22.      * done callback
  23.      */
  24.     it('should test a promise with return', () => {
  25.       return addPromise(1, 2).then(result => {
  26.         expect(result).to.equal(3);
  27.       });
  28.     });
  29.  
  30.     /**
  31.      * async await can also be used to test promises
  32.      */
  33.     it('should test promise with async await', async() => {
  34.       let result = await addPromise(1, 2);
  35.  
  36.       expect(result).to.equal(3);
  37.     });
  38.  
  39.     /**
  40.      * chai-as-promised plugin adds the 'eventually'
  41.      * matcher which can be used when testing a promise
  42.      */
  43.     it('should test promise with chai as promised', () => {
  44.       expect(addPromise(1, 2)).to.eventually.equal(3);
  45.     });
  46.   });
  47. });
Using sinon spies

Here, I tested a function called 'foo' in the example script(src/lib/demo.js). To install a spy to the context function, I first installed sinon(npm install --save-dev sinon) which is a library that provides test double(spies, stubs and mocks) for our unit tests. To use sinon, I also installed the sinon-chai(npm install --save-dev sinon-chai) plugin which would extend chai with assertions for sinon. 

  1. exports.foo = () => {
  2.     // some operation
  3.     console.log('console.log was called');
  4.     console.warn('console.warn was called');
  5.  
  6.     return;
  7. }
  1. const sinon = require('sinon');
  2. const sinonChai = require('sinon-chai');
  3.  
  4. /**
  5.  * Plugins are extended into chai using the .use() method once imported
  6.  */
  7. chai.use(sinonChai);
  8.  
  9. describe('demo.js', () => {
  10.   context('foo()', () => {
  11.     it('should spy on log', () => {
  12.       /**
  13.        * Here, we install a spy on the global console function
  14.        * which we will use to check if its method 'log' is invoked
  15.        */
  16.       const spy = sinon.spy(console, 'log');
  17.       foo();
  18.  
  19.       /**
  20.        * Here, 'calledOnce' is a spy strategy that checks if the function
  21.        * passed in the spy above is invoked at least once
  22.        */
  23.       expect(spy.calledOnce).to.be.true;
  24.       expect(spy).to.have.been.calledOnce;
  25.  
  26.       /**
  27.        * It is recommended to call the restore strategy so as to reset to the
  28.        * original method's state. This prevents unwanted behaviour when we install another spy to the
  29.        * same method again
  30.        */
  31.       spy.restore();
  32.     });
  33.   });
  34. });
Using sinon stubs

Here, I installed a stub on console.warn in the context function. Stubs are similar to spies with the main difference being they do not actually invoke the methods being tested but simulate the invokation. 

  1. exports.foo = () => {
  2.     // some operation
  3.     console.log('console.log was called');
  4.     console.warn('console.warn was called');
  5.  
  6.     return;
  7. }
  1. describe('demo.js', () => {
  2.   context('foo()', () => {
  3.     it('should stub console.warn', () => {
  4.       /**
  5.        * Here, callsFake is used to simulate an output of an example
  6.        * stub installed. We can use this in a case where we are stubbing
  7.        * insertion to a database and we want to return back a fake object as output
  8.        */
  9.       let stub = sinon.stub(console, 'warn').callsFake(() => {
  10.         console.log('message from stub');
  11.       });
  12.  
  13.       foo();
  14.  
  15.       expect(stub).to.have.been.calledOnce;
  16.       expect(stub).to.have.been.calledWith('console.warn was called');
  17.  
  18.       stub.restore();
  19.     });
  20.   });
  21. });
Testing private code & rewire

Here, I used a dependency called Rewire which lets you access private function within a file for better unit testing. The library provides getters and setters that can be used to access these functions/variables. In this example, I used a setter to replace a private function with a stub so as to simulate invokation of the function. Installation: npm install --save-dev rewire 

  1. exports.bar = async(file_name) => {
  2.   await exports.createFile(file_name);
  3.   let result = await callDb(file_name);
  4.  
  5.   return result;
  6. }
  7.  
  8. exports.createFile = (file_name) => {
  9.   console.log('---in createFile');
  10.  
  11.   return new Promise((resolve) => {
  12.     setTimeout(() => {
  13.       console.log('fake file created');
  14.  
  15.       resolve('done');
  16.     }, 20);
  17.   });
  18. }
  19.  
  20. const callDb = (file_name) => {
  21.   console.log('---in callDb');
  22.  
  23.   return new Promise((resolve) => {
  24.     setTimeout(() => {
  25.       console.log('fake db call');
  26.  
  27.       resolve('saved');
  28.     }, 20);
  29.   });
  30. }
  1. /**
  2.  * To use rewire you must swap with 'require' when importing the file.
  3.  * This allows rewire to pass the getters and setters
  4.  */
  5. const demo = rewire('../../lib/demo');
  6.  
  7. describe('demo.js', () => {
  8.   context('stub private functions', () => {
  9.     it('should stub createFile()', async() => {
  10.       let create_stub = sinon.stub(demo, 'createFile').resolves('create_stub');
  11.       let call_db = sinon.stub().resolves('call_db_stub');
  12.  
  13.       /**
  14.        * rewire provides getters and setters for the private variables in
  15.        * the file being tested.
  16.        * below the first parameter is the private function/variable being
  17.        * set and the second is the stub replacing it.
  18.        */
  19.       demo.__set__('callDb', call_db);
  20.  
  21.       let result = await bar('test.txt');
  22.  
  23.       expect(result).to.equal('call_db_stub');
  24.       expect(create_stub).to.have.been.calledOnce;
  25.       expect(create_stub).to.have.been.calledWith('test.txt');
  26.       expect(call_db).to.have.been.calledOnce;
  27.  
  28.       create_stub.restore();
  29.     });
  30.   });
  31. });
Run Setup

First we setup a script in the package.json file like so: 

  1. "scripts": {
  2.     "test": "cross-env NODE_ENV=development mocha test/**/*.spec.js"
  3.  }

To run the tests above, enter the command: 

  1. npm test
